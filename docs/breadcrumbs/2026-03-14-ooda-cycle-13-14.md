## Cycle 13: 100% API Enforcement Coverage

### Problem
4 of 14 API routes lacked thermosolve+cbfCheck enforcement:
- `/api/warhammer/products` — product search
- `/api/warhammer/prices` — price lookup
- `/api/warhammer/optimize` — army optimization
- `/api/memory` — conversation/memory CRUD (GET + POST)

### Solution
Added `thermosolve()` + `cbfCheck()` enforcement to all 4 routes. Each route now:
1. Computes thermosolve signature from request context
2. Runs all 9 CBF barriers
3. Returns 403 with barrier failure details if blocked
4. Proceeds to business logic only if all barriers pass

### Result
**14/14 API routes now enforced — 100% coverage.**

## Cycle 14: CBF Barrier Visualization in Admin Dashboard

### Problem
Admin dashboard showed `CBF: ALL 8 SAFE` as a single text line — no per-barrier detail, and the count was wrong (9 barriers since FEP was added in v13.0).

### Solution
1. Enhanced `/api/admin/stats` to return full per-barrier results (name, safe, value) for all 9 barriers
2. Replaced single-line CBF text with a 3x3 grid showing each barrier:
   - BNR (Truth), BNN (Naturality), BNA (Energy), TSE (Thermal), PCD (Coherence)
   - OGP (Error), ECM (Quality), SPC (Synergy), FEP (Free Energy)
3. Each barrier shows green/red coloring based on safe status, plus the numeric value
4. Updated `DashboardData` interface to accept barrier details
